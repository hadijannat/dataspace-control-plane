# Handoff Summary

## When to use
- Use at the end of a task or before handing work back to a lead or neighboring owner.

## Steps
1. State the owned scope completed.
2. List files changed.
3. List verification run and what could not be run.
4. Record blockers, dependency notes, and follow-up needed from other owners.
5. End with concrete next steps for the lead or next specialist.

## Output
- A short handoff note with:
  - scope
  - files changed
  - tests run
  - blockers
  - next steps
